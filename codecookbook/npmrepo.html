Overview

  1. Create remote repository and clone locally
  2. Initialize NPM for repository (generate a package.json file)

1. Initialize Repository

a. Determine initial repository settings

Repository Name
Name of repository you'll be creating.
Description
Optional (but recommended) - Short description of the new repository.
Add a license
Optional (but recommended) - Typically, you would choose an open source license to apply the contents of the repository.
Generally, in Git repositories, a LICENSE file will contain the actual terms of the license for the contents of the repository. The package.json file that will be generated will also contain an identifier string that indicates the license to be applied to the contents.

The default license identifier for NPM packages is ISC, which represents the Open Source Initiative license. However (as of the time this document was created) GitHub does not have an option for automatically adding a LICENSE file for an Open Source Initiative license.

One option would be to select GNU General Public License v3.0 when you create the remote repository, and then entering GPL-3.0 when you initialize the repository for NPM.

If you wish to use ISC as the license for the package.json file, see https://opensource.org/licenses/ISC for a template wording to use in a corresponding LICENSE file. Verbiage for additional open source license terms can be viewed at https://opensource.org/licenses

b. Create new remote GitHub repository

Use the pre-determined repository settings from the previous step, along with the other following options:
Initialize this repository with a README
Optional (but recommended) - Adds README.md to the repository.
Add .gitignore
Select Node.

c. Make local clone of remote repository

From the terminal window or a git bash/cmd prompt, change to the subdirectory which is going to be the parent folder of the local repository, and clone the remote repository. 
cd parentDirectory
git clone --recurse-submodules repository_url
A subdirectory will be created which is the same name as the remote repository.

2. Initialize NPM

Open the terminal windows and change to the subdirectory of the local repository, then execute the NPM initialization command. 
cd path_to_repository
npm init
Answer the prompts as follows:
package name
Typically, you can just hit enter and accept the defaults, which will probably the same as the repository name in all lower-case letters.
version
You can accept the default or enter a different version number, depending upon your preference.
description
You can use the same description you used when you created the remote repository.
entry point
It is usually okay to accept the default value, and won't break anything. This may change, depending upon what is going to be in the NPM package.
test command
This can be left blank, unless you already know what to enter here.
git repository
NPM should be able to guess the default value correctly (from the git settings). Otherwise, enter the URL you used when you cloned the remote repository.
keywords
If is okay to leave this initially blank.
author
Name of the author (presumably, yourself).
license
This is an abbreviated string which represents the license associated with the NPM package. If you added a LICENSE file when you created the remote repository, then this should correspond with that license.
